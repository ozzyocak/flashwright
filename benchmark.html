Why flashwright
When to use it instead of Playwright
⚡

Repeated commands

CLI scraping, CI/CD pipelines, dev workflows where you run browser commands multiple times. Daemon makes them instant.

📦

Minimal footprint

850KB binary vs 349MB. No Node.js, no npm. Ship it in a Docker image, embed it anywhere, forget about dependencies.

💻

Script automation

Simple script files, one command per line. No test framework, no async/await, no setup. Just run and go.

🚀

Fast form filling

CDP batch pipelining sends mouse events in one write. 1.5x faster than Playwright for form interactions.

⚙️

Shell scripting

Pipe-friendly CLI. Use in bash scripts, Makefiles, cron jobs. No need to write a JS file for a 3-step browser task.

🧰

100% Rust

Memory-safe, zero-cost abstractions, compiled to native code. No garbage collection, no JIT warmup, no runtime overhead.

Honest limitations
When Playwright is still the better choice
🔴

Cross-browser testing

Playwright supports Chromium, Firefox, and WebKit. flashwright is Chromium-only.

🔴

Complex test suites

Playwright has assertions, locators, auto-waiting, parallel test runners. flashwright is a CLI, not a test framework.

🔴

Single one-shot commands

For a single cold-start eval, Playwright's Node module caching makes it faster (0.44s vs 1.57s).
